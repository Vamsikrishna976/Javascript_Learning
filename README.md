# Javascript Learning

This repository is a personal learning playground for JavaScript. It contains example code, small exercises, and mini projects that demonstrate the core language concepts from basics through more advanced topics.

## ЁЯУБ Repository Structure

- `index1.html` / `practice.js` тАФ quick test & sandbox files for trying out snippets.
- `Day_1_Implimentation/` тЖТ basic console logging and function examples.
- `Day_2_Variables/` тЖТ variable declaration and usage.
- `Day_3_var,let,const/` тЖТ difference between `var`, `let`, and `const`.
- `Day_4_Data_types/` тЖТ primitive and reference type examples.
- `Day_5_Template_Literal/` тЖТ template literal usage.
- `Day_6_Operators/` тЖТ arithmetic, assignment, comparison, logical, bitwise, ternary, and unary operator examples.
- `Day_7_Conditional_Statements/` тЖТ `if`, `if/else`, `if/else if/else`, and `switch` statements.
- `Day_8_Loops/` тЖТ `for`, `while`, `do/while` loop examples.
- `Day_9_Arrays/` тЖТ baseline array usage plus array method examples.
- `Day_10_Functions/` тЖТ function declarations, expressions, and basic patterns.
- `Day_11_Function_types/` тЖТ function types, arrow functions, callbacks.
- `Day_12_Scope/` тЖТ scope, hoisting, closures.
- `Day_13_parameters/` тЖТ function parameters, default values, rest/spread.
- `Day_14_DOM/` тЖТ DOM manipulation and event handling examples.
- `Day_14.1_Hoisting/` тЖТ JavaScript hoisting behavior with `var`, `let`, and `const`.
- `Day_15_Map/` тЖТ working with the `Map` data structure.
- `Day_16_CallbackFunctions/` тЖТ callback functions, async patterns, function passing.
- `Day_17_Scope,ScopeChain&LaxicalEnvirment/` тЖТ scope chain, lexical environment, and closure concepts.
- `Day_18_TemporalDeadZone/` тЖТ temporal dead zone, `let`/`const` behavior before initialization.

## ЁЯЪА Getting Started

1. **Open the folder** in a code editor (e.g., VS Code).
2. Open an HTML file (e.g., `Day_14_DOM/index.html`) in your browser to run UI examples.
3. Run standalone Node scripts (e.g., `practice.js`) via terminal:

```sh
node practice.js
```

## ЁЯзк How to Use

- Explore the numbered `Day_*` folders in order to follow a gradual learning path.
- Open the `.js` files to read the code and add your own experiments.
- Open the `.html` files in your browser to see DOM-related examples.

## тЬЕ Recommended Workflow

1. Pick a day/topic folder.
2. Read the existing code and comments.
3. Modify the code or add a new example file.
4. Run the code (Node for `.js`, browser for `.html`).
